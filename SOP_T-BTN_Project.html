Standard Operating Procedure

T-BTN Army Dog Health Telemetry

IIT Jodhpur Research Prototype

Field Deployment & Operational Guidelines

1. Purpose & Scope

  • Objective: To monitor real-time physiological vitals (BPM, SpO2, Temperature) and geolocation of military working dogs during active patrols.
  • Scope: Covers pre-deployment hardware checks, sensor attachment, dashboard monitoring, and emergency protocols.
  • Target Audience: Handlers, Veterinary Officers, and Field Technicians.

2. System Components

Hardware (Edge)

  • Raspberry Pi 5
  • MAX30102 (Heart Rate/SpO2)
  • MLX90614 (Body Temperature)
  • NEO-6M (GPS Module)
  • 20W USB Powerbank
  • 4G LTE Dongle

Software (Cloud)

  • SQLite WAL (Local Buffer)
  • Firebase Firestore (Database)
  • Next.js Web Dashboard
  • Automated Alerting System

3. Pre-Deployment Checks

Before leaving the base, the technician must verify:

  • Power: Ensure the 20W USB Powerbank is at 100% capacity.
  • Sensors: Verify I2C cables (MAX30102 on Bus 1, MLX90614 on Bus 3) and UART pins (GPS) are securely connected.
  • Connectivity: Ensure the 4G LTE dongle has an active SIM card with a data plan.
  • Collar: Inspect the dog collar harness for any loose wires or exposed electronics.

4. Hardware Boot Sequence

  1. Connect the Raspberry Pi 5 to the powerbank.
  2. Observe the Pi's red PWR LED (solid) and green ACT LED (blinking).
  3. Wait ~60 seconds for the Linux OS and tbtn.service systemd daemon to start.
  4. The system automatically initializes sensors and begins buffering data locally to SQLite.
  5. Wait for the 4G dongle LED to indicate an active cellular connection (usually solid blue/green).

5. Sensor Attachment (K9)

Proper placement is critical for accurate readings:

  • MAX30102 (Pulse Ox): Attach to a hairless or shaved patch of skin (e.g., ear flap, groin, or specifically designed collar contact point). Keep it shielded from direct sunlight.
  • MLX90614 (Temp): Position the IR sensor facing the skin surface, approximately 1-2 cm away. Keep the lens clean.
  • NEO-6M (GPS): Ensure the ceramic antenna faces the sky on the upper harness, free from metallic obstructions.

6. Dashboard Monitoring

For the remote command center or veterinary officer:

  • Navigate to the secure Vercel deployment URL.
  • Use the Device Selector (top right) to select the correct Dog ID (e.g., tbtn-001).
  • Verify the Status Badge shows LIVE (updates every ~15 seconds).
  • Confirm GPS fix is obtained on the Leaflet map.

7. Handling Critical Alerts

If vitals cross dangerous thresholds (e.g., BPM > 160, SpO2 < 90%):

  • The dashboard will flash a pulsing red border.
  • An audio alarm will sound, and a browser notification will appear.
  • Action: Radio the handler immediately. Halt the patrol, rest the dog, and assess for heatstroke or trauma.
  • Note: Readings will remain marked as "Critical" until vitals return to normal ranges.

8. Operating in Dead Zones

The system is designed for zero data loss:

  • If cellular signal is lost, the Status Badge on the dashboard will turn OFFLINE / STALE.
  • The Pi will continue polling sensors and saving to the local SQLite database.
  • Action: No action required. When the dog returns to network coverage, the firebase_client.py daemon will automatically flush all buffered data to the cloud.

9. Emergency Troubleshooting

  • Dashboard Frozen: Refresh the browser. Check if the device is simply out of range.
  • I2C Bus Hangs: The Pi 5 is equipped with a hardware Watchdog. If the sensor polling loop deadlocks, the Watchdog will automatically forcefully reboot the Pi within 15 seconds.
  • Undertension Warning: If the dashboard shows a throttle flag, replace the USB powerbank immediately (voltage dropping below 4.6V).
